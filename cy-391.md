## 项目主要需求

### 项目概述
**项目类型：Web前端**
搭建一个个人碳足迹可视化工具，用户记录日常出行、饮食、用电等行为，系统计算碳排放量并以图表展示，倡导低碳生活。

### 功能需求

1. **碳排放记录**：按类别记录日常行为数据——出行（公交/地铁/自驾/骑行/步行，公里数）、饮食（素食/混合/肉食为主，餐数）、用电（度数）、购物（消费金额）。
2. **日/周/月碳排放图表**：折线图展示碳排放量时间趋势，堆叠柱状图展示各品类排放占比。
3. **碳排放等级评估**：根据月度碳排放总量评定等级（低碳达人/合格/偏高/严重偏高），与全国人均碳排放对比。
4. **减碳建议**：根据排放结构给出个性化减碳建议（如"本月自驾出行占比较高，建议改为公共交通"）。
5. **碳积分系统**：低碳行为积累碳积分，积分可兑换虚拟勋章（种树徽章/骑行达人/素食主义者等）。
6. **排行榜**：本地排行榜（浏览器存储），展示近 30 天碳排放最低的记录，激励减碳。
7. **数据导入导出**：支持导出碳排放记录为 CSV，支持从 CSV 导入历史数据。

### 技术要求

- 前端框架：React 18 + TypeScript
- UI 库：Tailwind CSS + shadcn/ui
- 图表：Recharts（折线图、柱状图、饼图）
- 状态管理：Zustand
- 构建工具：Vite
- 数据持久化：浏览器 IndexedDB
- 代码规范：严格遵循单一职责原则，每个文件只负责一个功能；常量（排放因子系数、等级阈值、勋章条件）统一维护在 `src/constants/` 目录下；类型定义统一在 `src/types/`；所有工具函数封装在 `src/utils/`。

## 代码存放规则

这是纯前端单页应用（SPA）项目，无后端、无数据库。

前端代码统一放在 `frontend/`：

```
frontend/
├── src/
│   ├── pages/              # 页面组件
│   ├── components/         # 通用组件
│   ├── hooks/              # 自定义 Hooks
│   ├── stores/             # Zustand 状态管理
│   ├── storage/            # IndexedDB 封装
│   ├── constants/          # 常量定义（排放因子、等级阈值）
│   ├── types/              # TypeScript 类型
│   ├── utils/              # 工具函数（碳计算算法等）
│   ├── styles/             # 全局样式
│   └── App.tsx
├── public/
└── package.json
```
